✈ SkyBook API — Zadanie testowe QA

Kontekst

SkyBook to REST API systemu rezerwacji lotów i zarządzania biletami lotniczymi. API obsługuje:

  • Zarządzanie lotami (CRUD)
  • Rezerwacje z pasażerami, miejscami i płatnościami
  • Baza lotnisk
  • Przeliczanie walut

Twoje zadanie 30 min

  1. Zapoznaj się z API korzystając z dostępnych narzędzi
  2. Przetestuj API pod kątem poprawności działania — szczególnie pod kątem poniższych reguł biznesowych
  3. Zaraportuj znalezione błędy
💡 Wskazówka: REST Client ma 12 gotowych requestów w panelu bocznym — możesz je wysyłać od razu lub modyfikować. Nie musisz pisać JSON od zera.

Reguły biznesowe

#RegułaOpis
BR-01Numer lotuWymagany, unikalny, format: 2–3 litery + - + 1–5 cyfr (np. SB-1234)
BR-02Segmenty lotuMin. 1 wymagany; lot bezpośredni = 1 segment, z przesiadką = 2+
BR-03Segment: departure/arrivalairportCode – IATA 3 litery; dateTime – ISO 8601; arrival po departure; durationMinutes > 0
BR-04Segment: aircraftObiekt wymagany; model 1–100 znaków; seatConfiguration – wartości ≥ 0
BR-05Cena bazowa i walutabaseFare ≥ 0.00, max 2 dec; currency ∈ {USD, EUR, GBP, PLN}
BR-06PodatkiOpcjonalne; code 2 wielkie litery, unikalny; amount ≥ 0.00
BR-07OpłatyOpcjonalne; feeCode unikalny; amount ≥ 0.00; max 10 opłat
BR-08Suma cenyAuto: baseFare + Σtaxes + Σfees − discount; wartość musi być poprawna
BR-09RabatOpcjonalny; percentage 0–100; validUntil nie w przeszłości
BR-10Dostępne miejscaZmniejszane przy rezerwacji, przywracane przy anulowaniu
BR-11PasażerowieMin. 1, max 9 na rezerwację
BR-12Dane pasażerafirstName/lastName 1–100 zn; dateOfBirth ISO; email poprawny; phone międzynarodowy
BR-13Przydział miejscaseatNumber unikalny w locie; class ∈ {ECONOMY, BUSINESS, FIRST}; type ∈ {WINDOW, MIDDLE, AISLE}
BR-14PłatnośćWymagana; method ∈ {CREDIT_CARD, DEBIT_CARD, BANK_TRANSFER, BLIK}; cardDetails wymagane dla kart
BR-15Status rezerwacjiPENDING → CONFIRMED, PENDING → CANCELLED, CONFIRMED → CANCELLED; brak cofania
BR-16Usunięcie lotuNie można usunąć lotu z rezerwacjami
BR-17Przeliczanie walutPoprawne przeliczanie między USD, EUR, GBP, PLN
BR-18PaginacjaDomyślnie: page=0, size=10, max size=100
BR-19Wyszukiwanie lotówCase-insensitive, partial match
BR-20HTTP Status CodesPOST → 201, DELETE → 204, Not Found → 404, Validation → 400

PoleOpis

Oczekiwany rezultatCo powinno się stać wg reguł biznesowych

SeverityCritical / Major / Minor
Przykład raportu:

Tytuł: GET /api/v1/flights zwraca 500 zamiast pustej listy gdy brak lotów

Kroki reprodukcji:
1. Usuń wszystkie loty
2. Wyślij GET /api/v1/flights

Oczekiwany rezultat: 200 OK z pustą listą { "content": [], "totalElements": 0 }

Faktyczny rezultat: 500 Internal Server Error

Severity: Major

Dane testowe

API posiada wstępnie załadowane dane:

  • 6 lotnisk: WAW, FRA, LHR, JFK, CDG, BCN
  • 5 lotów: SB-1234, SB-5678, SB-9012, SB-3456, SB-7890
  • 3 rezerwacje (statusy: CONFIRMED, PENDING, CANCELLED)
  • Waluty: EUR (bazowa), USD, GBP, PLN
ℹ Uwaga: Baza danych jest resetowana przy każdym restarcie API.
